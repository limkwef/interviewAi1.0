## 1. 项目概述

### 1.1 项目背景
传统面试准备方式存在场景难以模拟、反馈缺乏针对性、练习效率低下等痛点。本项目旨在构建一款**AI 驱动的智能模拟面试系统**，通过大语言模型（DeepSeek）模拟真实面试官，为求职者提供岗位定制、回合制面试、多维评分、智能诊断的一站式面试训练平台。

### 1.2 产品定位
面向 IT 技术岗位求职者的 AI 模拟面试训练平台，覆盖以下角色：

| 角色 | 说明 |
|------|------|
| **普通用户** | 注册登录后可进行模拟面试、查看报告、管理错题、制定学习计划 |
| **管理员** | 用户管理、题库管理、反馈处理、操作审计 |

### 1.3 技术栈

| 层次 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + Vue Router 4 + Pinia + Element Plus 2.7 + ECharts 5.5 + Vite |
| 后端 | Spring Boot 3.5 + Java 21 + MyBatis + JWT (jjwt 0.12.6) |
| 数据库 | MySQL 8 (utf8mb4) + Redis 8 (缓存与分布式锁) |
| AI | DeepSeek Chat API (结构化输出 + SSE 流式) |
| 语音 | 百度语音 ASR + TTS API |
| 其他 | Maven, Lombok, Jakarta Validation, Spring Security Crypto |

### 3.4 AI 模拟面试（核心功能）

#### 3.4.1 面试配置

| 参数 | 可选值 |
|------|--------|
| 目标岗位 | Java 后端 / 前端 / 全栈 / 算法 |
| 面试轮次 | 技术面 / HR 面 / 综合面 |
| 难度 | 简单 / 中等 / 困难 |
| 题目数量 | 3-10（受题库数量动态上限约束） |

#### 3.4.2 面试流程

```
配置面试 → AI 开场问候 → 逐题提问 → 用户作答 → AI 评价/追问 → 
循环至最后一题 → 结束面试 → 生成评分报告 → 自动导入错题 → 自动生成诊断
```

| 环节 | 说明 |
|------|------|
| **AI 出题** | 根据岗位+轮次+难度组合，从题库中按比例选取题目 |
| **AI 追问** | 用户答完后 AI 可追问 1-2 轮深挖回答深度 |
| **流式响应** | SSE 实时流式推送 AI 回复（三阶段降级：SSE → 轮询 → POST） |
| **限时作答** | 每题记录作答时长（秒），报告展示耗时 |
| **终止退出** | 未作答的面试可放弃删除，已作答不可删除 |

#### 3.4.3 异常处理

- **断网重连**：前端检测连接断开后自动重试（最多 3 次）
- **SSE 降级**：SSE 连接失败 → 轮询模式 → 普通 POST
- **AI 熔断**：连续 5 次 AI 调用失败 → 熔断 30s → 半开恢复
- **空响应兜底**：AI 返回为空时使用本地评分引擎生成评分

### 3.5 评分系统

#### 3.5.1 五维评分模型

| 维度 | 说明 | 技术面权重 | HR 面权重 | 综合面权重 |
|------|------|-----------|-----------|-----------|
| 技术能力 | 知识点准确性、深度 | 40% | 10% | 30% |
| 表达能力 | 语言组织、条理性 | 15% | 35% | 20% |
| 逻辑思维 | 逻辑链条、结构化程度 | 20% | 15% | 20% |
| 完整度 | 回答覆盖范围 | 15% | 20% | 15% |
| 创新性 | 独立思考、拓展延伸 | 10% | 20% | 15% |

#### 3.5.2 总分计算

```
总分 = 各维度加权和 × 难度系数
难度系数：简单(0.85) / 中等(1.0) / 困难(1.15)
```

#### 3.5.3 等级划分

| 等级 | 分数范围 |
|------|---------|
| 优秀 | ≥ 90 |
| 良好 | 80-89 |
| 中等 | 70-79 |
| 待提高 | 60-69 |
| 不合格 | < 60 |

## 4. 非功能需求

### 4.1 性能指标

| 指标 | 目标值 |
|------|--------|
| 页面加载 | 首屏 < 2s |
| AI 响应 | 首字返回 < 3s（SSE 流式） |
| 接口响应（非 AI） | < 500ms（含缓存） |
| 并发面试 | 支持 50 用户同时面试 |

### 4.2 缓存策略

| 缓存对象 | TTL | 说明 |
|----------|-----|------|
| 题目列表 | 10min | 管理员新增题目后清空 |
| 看板数据 | 5min | 每次面试结束后清除 |
| 用户信息 | 1h | 修改后手动清除 |
| 诊断报告 | 2h | 生成后缓存复用 |
| 竞技分析 | 1h | 主动刷新或新报告触发 |

### 4.3 安全性

| 要求 | 措施 |
|------|------|
| 密码安全 | BCrypt 加密存储，无明文 |
| 接口鉴权 | JWT 令牌校验，24h 过期 |
| 角色隔离 | AdminInterceptor 校验 admin 角色 |
| 操作审计 | 所有管理操作记录入库（含 IP） |
| 文件上传 | 限制 2MB，仅图片格式 |
| 防重复提交 | Redis 分布式锁（面试创建接口） |

### 4.4 AI 容错

| 故障场景 | 处理方式 |
|----------|----------|
| AI API 超时 | 3 次重试（1s/2s 退避） |
| AI 连续失败 | 熔断 30s，使用本地评分引擎 |
| AI 返回非 JSON | 结构化解构失败 → 正则解析 → 兜底模板 |
| SSE 中断 | 前端断线重连 → 轮询 → POST 降级 |

### 4.5 前端兼容

- 支持现代浏览器（Chrome/Firefox/Edge/Safari）
- 录音依赖 MediaRecorder API，需 HTTPS 或 localhost
- 响应式布局适配 1366+ 宽屏

## 5. 数据流图

### 5.1 核心业务流程：模拟面试

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 面试配置  │───→│ AI 出题  │───→│ 用户作答  │───→│ AI 评分  │
│ 岗位/轮次  │    │ 题序打乱  │    │ 文字/语音  │    │ 追问/评价 │
│ 难度/数量  │    │ 去重分配  │    │ 计时记录  │    │ 流式推送  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                      │
                                                      ▼
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 学习路径  │←───│ AI 诊断  │←───│ 面试报告  │←───│ 最后一题  │
│ 分阶段任务 │    │ 深度分析  │    │ 五维评分  │    │ 结束面试  │
│ 进度追踪  │    │ 学习计划  │    │ 每题评语  │    │ 自动提交  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 5.2 数据流：报告生成

```
面试结束 → ScoringEngine 计算总分/维度分
         → ReportService 持久化报告
         → MistakeService 自动导入低分题（≤3分）
         → AIDiagnosisService 生成诊断报告
         → LearningPathService 更新学习路径（可选刷新）
         → CacheService 清除看板/竞技分析缓存
```

## 6. 需求优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0** | 用户认证、模拟面试核心流程、AI 评分 | MVP 不可缺 |

## 7. 边界条件与约束

| 约束 | 说明 |
|------|------|
| 题库依赖 | 面试出题要求各类别题库不少于 `题目数 × 类数` 条 |
| AI 依赖 | 系统正常运行依赖 DeepSeek API 可用性 |
| 语音环境 | 录音需用户浏览器授权麦克风，HTTPS 部署 |
| 岗位映射 | 仅支持 Java 后端 / 前端 / 全栈 / 算法 四个岗位 |
| 面试防重 | 同一用户不可并发创建面试（分布式锁） |
| 题目去重 | 同用户 30 天内已面试题目不会再次出现 |
