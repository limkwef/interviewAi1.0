## 1. 项目背景

AI 模拟面试系统目前面试模块的**时间感知**是空白：

| 现有信息 | 状态 | 说明 |
|----------|------|------|
| 题号进度 | ✅ 已有 | 第 3/5 题 |
| 总面试时长 | ❌ 无 | 用户不知道面了多久 |
| 每题耗时 | ❌ 无 | 用户不知道一道题想了多久 |
| 时间维度的报告 | ❌ 无 | 无法分析"答错的题是否花了更少时间" |

两位同学在不同场景下的对比：

| | A 同学 | B 同学 |
|--|--------|--------|
| 每题耗时 | 1-2 分钟 | 4-5 分钟 |
| 得分 | 中等 | 较低 |
| 行为推测 | 快速作答，想到就说 | 犹豫太久，时间压力大 |

如果没有计时数据，这种有价值的分析就无法实现。

### 1.1 为什么选择**正向**计时而非倒计时

| 方案 | 适用场景 | 选择 |
|------|----------|:----:|
| **正向计时**（总用时/每题用时） | 训练/自学模式，让用户了解自己的节奏 | ✅ **采用** |
| 倒计时（每题限时） | 压力面试模拟，增加紧迫感 | ❌ **不采用** |

正向计时更适合本项目定位（**面试训练工具**而非面试压力测试）：

1. **训练工具的第一目标是帮助用户成长**，不需要额外制造焦虑
2. 用户应该**先把回答质量做起来**，再去控制时间
3. 正向计时提供了**数据参考**（"我一般每题花 2 分钟"），倒计时提供了**压力**（"你只剩 30 秒"）

## 2. 需求分析

### 2.1 功能需求

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-01 | **面试总用时** | 面试页面顶部显示从创建到当前的累计时间 | P0 |
| F-02 | **当前题目用时** | 当前正在回答的题目已耗时，切换题目时归零 | P0 |
| F-03 | **每题耗时记录** | 面试结束时，将每题耗时持久化到数据库 | P0 |
| F-04 | 报告展示每题耗时 | 面试报告详情页展示每道题的用时 | P1 |
| F-05 | 面试历史列表展示总用时 | 历史记录列表显示每场面试的总时长 | P1 |
| F-06 | 时间维度的能力分析 | 后续仪表盘可展示"平均每题用时趋势" | P2 |

### 2.2 非功能需求

| 编号 | 需求 | 指标 |
|------|------|------|
| N-01 | 计时精度 | 秒级即可，不需要毫秒 |
| N-02 | 前端计时误差 | 离开页面后不计时（`beforeunload` 暂停），回到页面续接 |
| N-03 | 数据准确性 | 以服务器记录的 `created_at` 时间戳为准，前端仅做显示 |
| N-04 | 性能影响 | 计时逻辑不增加任何 API 调用次数 |

### 2.3 边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户刷新页面 | 前端从服务端获取 session 的 `startedAt`，重新开始计时 |
| 用户长时间离开（吃饭/接电话） | 前端 `beforeunload` 暂停计时，回到页面续接（总计时含离开时间） |
| 切换到其他标签页 | `visibilitychange` 暂停/恢复计时，总计时**不含**不可见时间 |
| 面试中途放弃 | 不记录任何时间数据，正常删除 session |
| 追问场景 | **追问归入当前题计时**，不单独计时间片 |

## 3. 设计原则

### 3.1 哲学：计时器是"信息"而非"压力"

```
┌─────────────────────────────────────────────────────┐
│  计时器的设计定位：提供数据参考，而非制造焦虑            │
│                                                       │
│  正确的做法：                                         │
│    ✅ 字号小、颜色淡，融于界面                          │
│    ✅ 不闪烁、不跳动、不变色                            │
│    ✅ 放在标题栏信息区，而非正中央                       │
│    ❌ 不要做成红色倒计时                                │
│    ❌ 不要弹出"你已经用了 X 分钟"的提示                  │
└──────────────────────────────────────────────────────┘
```

### 3.2 时间计算规则

```
总面试用时 = 当前时间 - session.created_at

当前题用时 = 当前时间 - 该题的首次 AI 提问的消息时间
            （消息表中 role=ai AND messageType=question 的 created_at）

每题耗时 = 下一题开始时间 - 本题开始时间
         （最后一道题 = 面试结束时间 - 本题开始时间）
```

### 3.3 前端计时模型

```
┌──────────────────────────────────────────────────────────┐
│  前端计时器（纯客户端，不依赖服务器同步）                   │
│                                                           │
│  onMounted → 记录 startTimestamp                          │
│  setInterval(1s) → elapsed = now - startTimestamp          │
│                                                           │
│  visibilitychange(hidden) → pause (记录 pausedAt)         │
│  visibilitychange(visible) → resume (累加 pauseDuration)  │
│                                                           │
│  跳题事件 (questionIndex 变化)                             │
│    → 计算本题用时 = now - questionStartTimestamp           │
│    → 存入 Map<questionIndex, duration>                     │
│    → 重置 questionStartTimestamp = now                     │
│                                                           │
│  面试结束事件 (endInterview)                                │
│    → 停止所有定时器                                        │
│    → 提交所有每题耗时到服务端                                │
└──────────────────────────────────────────────────────────┘
```

## 4. 数据模型设计

### 4.1 数据库变更

**最小方案：不新增表，在 `report_comment` 增加字段**

```java
/**
 * 保存每题耗时数据（更新 report_comment.duration_seconds）
 */
@Transactional
public void saveQuestionDurations(Long reportId, Long userId, 
                                   List<Map<String, Object>> durations) {
    // 1. 校验报告归属
    Map<String, Object> report = getReportDetail(reportId, userId);
    if (report == null) {
        throw new BusinessException(404, "报告不存在");
    }
    
    // 2. 按 sort_order 更新每题耗时
    for (Map<String, Object> item : durations) {
        Integer questionIndex = ((Number) item.get("questionIndex")).intValue();
        Integer seconds = ((Number) item.get("seconds")).intValue();
        // questionIndex 对应 report_comment 的 sort_order (1-based)
        commentMapper.updateDurationByReportIdAndOrder(reportId, questionIndex, seconds);
    }
    
    logger.info("报告{}已保存每题耗时数据，共{}条", reportId, durations.size());
}
```

**改动点 2：Script — 新增计时变量和逻辑**

```javascript
// ======================== 计时器逻辑 ========================

const startTimestamp = ref(0)                  // 面试开始（页面加载）时间戳
const questionStartTimestamp = ref(0)          // 当前题开始时间戳
const totalElapsed = ref(0)                    // 总用时（秒）
const currentQuestionElapsed = ref(0)          // 当前题用时（秒）
const questionDurations = ref({})              // Map<questionIndex, 秒>
const questionTimer = ref(null)                // 总计时器句柄
const pausedSince = ref(0)                     // 暂停时的时间戳
const totalPauseDuration = ref(0)              // 累计暂停时间（秒）
```

```javascript
// 启动计时器
function startTimers() {
  startTimestamp.value = Date.now()
  questionStartTimestamp.value = Date.now()
  
  questionTimer.value = setInterval(() => {
    if (!session.value) return
    
    const now = Date.now()
    // 总用时 = (当前时间 - 开始时间) - 累计暂停时间
    totalElapsed.value = Math.floor((now - startTimestamp.value) / 1000) - totalPauseDuration.value
    // 当前题用时 = (当前时间 - 本题开始时间) - 暂停时间（粗略）
    currentQuestionElapsed.value = Math.floor((now - questionStartTimestamp.value) / 1000) - totalPauseDuration.value
    
    if (currentQuestionElapsed.value < 0) currentQuestionElapsed.value = 0
    if (totalElapsed.value < 0) totalElapsed.value = 0
  }, 1000)
}

// 停止计时器
function stopTimers() {
  if (questionTimer.value) {
    clearInterval(questionTimer.value)
    questionTimer.value = null
  }
}

// 跳题时记录当前题用时
function recordQuestionDuration() {
  if (!session.value) return
  const qIndex = session.value.currentQuestion
  const elapsed = currentQuestionElapsed.value
  if (elapsed > 0) {
    questionDurations.value[qIndex] = elapsed
  }
  // 重置当前题计时
  questionStartTimestamp.value = Date.now()
  currentQuestionElapsed.value = 0
}

// 提交所有计时数据到服务端
async function submitDurations(reportId) {
  // 把最后一道题也记上
  if (session.value) {
    const lastQ = session.value.currentQuestion
    const lastElapsed = currentQuestionElapsed.value
    if (lastElapsed > 0) {
      questionDurations.value[lastQ] = lastElapsed
    }
  }
  
  const durations = Object.entries(questionDurations.value).map(([qIndex, seconds]) => ({
    questionIndex: parseInt(qIndex),
    seconds
  }))
  
  if (durations.length === 0) return
  
  try {
    await submitReportDurations(reportId, { durations })
  } catch (e) {
    console.warn('提交计时数据失败（不影响主流程）', e)
  }
}

// 格式化时长
function formatDuration(seconds) {
  if (!seconds || seconds < 0) return '0:00'
  const m = Math.floor(seconds / 60)
  const s = Math.floor(seconds % 60)
  return `${m}:${s.toString().padStart(2, '0')}`
}
```

**改动点 3：Script — 页面可见性处理**

```javascript
// 页面可见性变化时暂停/恢复计时
function handleVisibilityChange() {
  if (document.hidden) {
    // 页面隐藏 → 暂停
    pausedSince.value = Date.now()
    isPageVisible.value = false
  } else {
    // 页面恢复 → 累加暂停时间
    if (pausedSince.value > 0) {
      totalPauseDuration.value += Math.floor((Date.now() - pausedSince.value) / 1000)
      pausedSince.value = 0
    }
    isPageVisible.value = true
  }
}
```

```javascript
// 在 loadSessionInfo 成功后

  // 如果面试已进行了一段时间，从服务端时间算起
  const serverStart = new Date(res.data.createdAt).getTime()
  const elapsed = Math.floor((Date.now() - serverStart) / 1000)
  totalElapsed.value = Math.max(0, elapsed)
  // 调整 startTimestamp 使计时器接续
  startTimestamp.value = Date.now() - elapsed * 1000 + totalPauseDuration.value * 1000
```

**改动点 6：Script — 结束面试时提交计时数据**

在 `handleEndInterview` 成功后、跳转前：

```javascript
async function handleEndInterview() {
  ending.value = true
  try {
    const res = await endInterview(sessionId)
    if (res.code === 200) {
      stopTimers()
      interviewEnded.value = true
      setInterviewState(false)
      
      // 新增：提交计时数据（不阻塞跳转）
      if (res.data?.reportId) {
        submitDurations(res.data.reportId)
      }
      
      router.replace(`/report/detail/${res.data.reportId}`)
    }
  } catch (e) {
    ElMessage.error('结束面试失败')
  } finally {
    ending.value = false
  }
}
```

> ⚠️ 注意：`submitDurations` 用 fire-and-forget 方式，不 await，不阻塞页面跳转。即使提交失败也不影响用户体验。
