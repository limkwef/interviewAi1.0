## Phase 3：后端 sendMessage 去重（省 ~120 行）

### 问题

`InterviewService.java` 中 `sendMessage()`(73 行) 和 `sendMessageStream()`(134 行) 的前半部分完全重复：

```java
// ↓ 以下两段在 sendMessage 和 sendMessageStream 中各写了一遍 ↓

InterviewSession session = validateSession(sessionId, userId);
Question currentQuestion = getCurrentQuestion(session);
String currentQuestionText = currentQuestion != null ? currentQuestion.getTitle() : "";
List<Question> allQuestions = getSessionQuestions(session);

// 保存用户消息
InterviewMessage userMessage = new InterviewMessage();
userMessage.setSessionId(sessionId);
userMessage.setRole("user");
userMessage.setContent(content);
userMessage.setMessageType("answer");
userMessage.setQuestionIndex(session.getCurrentQuestion());
messageMapper.insert(userMessage);

// 加载上下文
List<Map<String, String>> history = loadContext(sessionId, session);

// ↓ 只有以下部分不同 ↓

// sendMessage: 调 evaluateAndRespond → 存一条 AI 消息 → 返回
// sendMessageStream: 调 chatStream → 流式推送 → 结束后再存 AI 消息
```

### 方案

提取公共前置步骤为私有方法：

```java
/**
 * 发送消息的前置公共逻辑：校验 → 查题 → 存用户消息 → 加载上下文
 * 让 sendMessage 和 sendMessageStream 共用
 */
private SendMessageContext prepareSendMessage(Long sessionId, Long userId, String content) {
    InterviewSession session = validateSession(sessionId, userId);
    Question currentQuestion = getCurrentQuestion(session);
    List<Question> allQuestions = getSessionQuestions(session);

    InterviewMessage userMessage = new InterviewMessage();
    userMessage.setSessionId(sessionId);
    userMessage.setRole("user");
    userMessage.setContent(content);
    userMessage.setMessageType("answer");
    userMessage.setQuestionIndex(session.getCurrentQuestion());
    messageMapper.insert(userMessage);

    List<Map<String, String>> history = loadContext(sessionId, session);

    SendMessageContext ctx = new SendMessageContext();
    ctx.setSession(session);
    ctx.setCurrentQuestion(currentQuestion);
    ctx.setAllQuestions(allQuestions);
    ctx.setHistory(history);
    ctx.setUserId(userId);
    return ctx;
}
```

同时新增一个内部类 `SendMessageContext`（直接放在 `InterviewService.java` 末尾，不单独一个文件）。

| 指标 | 改前 | 改后 |
|------|:----:|:----:|
| sendMessage 行数 | 73 | ~25 |
| sendMessageStream 行数 | 134 | ~85 |
| 新增 prepareSendMessage | 0 | ~35 |
| **总行数** | **207** | **145** |
| **节省** | | **~62 行**（剔除上下文类的净节省） |

public String buildScoringSystemPrompt(...) {

## Phase 6：前端 JS 统一（省 ~100 行）

### 6.2 Router 精简

**问题**：`router/index.js` (229 行) 中每个路由都完整写了 `path/component/meta`。

**方案**：提取路由配置的公共默认值。

```javascript
// 改前
{
    path: '/interview/config',
    name: 'InterviewConfig',
    component: () => import('@/views/interview/config/index.vue'),
    meta: { requiresAuth: true, title: '模拟面试配置' }
},
{
    path: '/interview/session/:id',
    name: 'InterviewSession',
    component: () => import('@/views/interview/session/index.vue'),
    meta: { requiresAuth: true, title: '模拟面试' }
},

// 改后：创建路由工厂
function route(path, name, component, title) {
    return {
        path,
        name,
        component: () => import(`@/views/${component}.vue`),
        meta: { requiresAuth: true, title }
    }
}
// 使用
route('/interview/config', 'InterviewConfig', 'interview/config/index', '模拟面试配置'),
route('/interview/session/:id', 'InterviewSession', 'interview/session/index', '模拟面试'),
```

**节省**：~50 行

## 7. 安全注意事项

| 操作 | 风险 | 预防 |
|------|------|------|
| 合并 VO 文件 | 其他文件 import 路径失效 | 用 IDE 的重构功能，批量替换 import |
| CSS 变量替换 | 漏改某个 hardcode 导致样式异常 | 替换后逐页截图对比 |
| 抽取公共组件 | 组件 props 变化影响现有页面 | 保持接口兼容，旧 props 标记 @deprecated |
| 合并 API 文件 | 前端 import 引用断裂 | 改完立即 `npm run build` 验证 |
| 合并 Service 方法 | 回归测试覆盖不全 | 重点验证面试全流程（创建→发消息→流式→结束） |
