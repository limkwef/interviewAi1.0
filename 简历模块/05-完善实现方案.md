# 完善实现方案（完整版）

## 1. 目标

在初级方案基础上，实现**上传→编辑→版本管理→简历驱动出题→技能验真**的完整闭环，让简历模块成为面试系统真正的个性化引擎。

## 2. 范围

| 新增能力 | 说明 |
|---|---|
| 双入口一体化 | 上传解析后直接进入编辑页确认/修改，而非分两条独立路径 |
| 版本管理与对比 | 保留所有历史版本，用户可以 diff 任意两版的变化 |
| 可配置简历模版 | 管理后台配置不同岗位方向的模版字段 |
| 技能分级出题 | 根据简历声称的熟练度级别，差异化选择题目深度 |
| 技能验证报告 | 面试后自动生成简历声称 vs 实际表现的对比 |
| 解析降级链路 | circuit breaker + 半结构化 regex 提取（复用现有 ScoringEngine 模式） |

## 3. 流程设计

### 3.1 一体化简历编辑流程

```
上传 PDF
   ↓ → AI 解析 → parsed_data ↓
                               ↓
   自动跳转到编辑页 ← ← ← ← ← ←
       │  表单预填充 AI 解析结果
       │  用户可以：
       │    - 修改任意字段
       │    - 增删技能/项目
       │    - 调整熟练度
       ▼
   确认保存 → INSERT 新版本 (source=template, is_active=1)
       ↓
   面试时自动携带此版本
```

### 3.2 版本管理流程

```
每次编辑/上传 → 新增一条 resume 记录，version+1
                      ↓
简历列表 → 显示所有版本（含来源/时间/是否活跃标记）
             → 点任一版本 → 详情
             → 点击"激活" → 此版本 is_active=1，其余 is_active=0
             → 勾选两版 → "对比" → 版本 diff 页
```

### 3.3 技能分级出题逻辑

在 `PromptBuilder` 中，简历技能不再只是文本列表，而是**分级出题指令**：

```
=== 简历技能分级出题策略 ===
准备就绪的技能（精通）→ 深度原理 + 架构设计题（难度：hard）
  示例：简历写"精通 Redis" → 出"Redis 哨兵 vs 集群 vs RedisShard 选型"

已具备的技能（熟练） → 常规原理 + 场景题（难度：medium）
  示例：简历写"熟练 Spring Boot" → 出"Spring Boot 自动配置原理"

有一定了解（熟悉） → 基础概念 + 使用题（难度：medium-easy）
  示例：简历写"熟悉 Docker" → 出"Docker 和虚拟机区别及常用命令"

仅接触过（了解） → 概念确认 + 适当追问（难度：easy）
  示例：简历写"了解 ES" → 出"ES 倒排索引是什么"
```

同时，`QuestionController` 中现有的按「方向+难度」选题目逻辑增加简历技能过滤：

```java
// 改进前：只按 direction + difficulty 随机选
List<Question> questions = questionMapper.selectByDirectionAndDifficulty(direction, difficulty);

// 改进后：优先选简历技能标签匹配的题目，选不够再用通用题库补
if (resumeSkills != null && !resumeSkills.isEmpty()) {
    questions = questionMapper.selectBySkills(resumeSkills, count);
    if (questions.size() < count) {
        // 补充通用题目
    }
}
```

## 4. 新增能力详解

### 4.1 版本对比

```java
@Service
public class ResumeCompareService {

    /**
     * 对比两个版本的简历，返回差异列表
     */
    public CompareResult compare(ResumeData oldVersion, ResumeData newVersion) {
        CompareResult result = new CompareResult();
        
        // 技能变化
        result.setSkillsAdded   (subtract(newSkills, oldSkills));
        result.setSkillsRemoved (subtract(oldSkills, newSkills));
        result.setSkillsUpgraded(findLevelUp(oldSkills, newSkills));   // 熟练度提升
        result.setSkillsDowngraded(findLevelDown(oldSkills, newSkills));
        
        // 项目变化
        result.setProjectsAdded   (subtract(newProjects, oldProjects));
        result.setProjectsRemoved (subtract(oldProjects, newProjects));
        
        // 工作经历变化
        result.setExperienceChanges(compareList(oldExp, newExp));
        
        return result;
    }
}
```

前端使用 diff 视图展示：

```
│ 新增: Docker(熟练) Kubernetes(了解)      │
│ 移除: RocketMQ                           │
│ 提升: Redis 了解→熟练   Java 熟练→精通   │
│ 下降: (无)                               │
└──────────────────────────────────────────┘
```

### 4.2 技能验证报告

面试结束后异步生成，作为诊断报告的一部分：

```java
@Service
public class SkillVerificationService {

    /**
     * 面试结束后调用，生成技能验证数据
     */
    public SkillVerificationReport verify(Long sessionId, ResumeData resumeData) {
        // 1. 从面试消息中提取所有问答对
        List<QAPair> qas = interviewMessageService.getQABySession(sessionId);
        
        // 2. 按技能维度归类
        Map<String, List<QAPair>> bySkill = classifyBySkill(qas);
        
        // 3. 对每个技能评估
        for (Map.Entry<String, List<QAPair>> entry : bySkill.entrySet()) {
            String skillName = entry.getKey();
            List<QAPair> skillQAs = entry.getValue();
            
            // 调用 AI 评估该技能的实际水平
            String verifiedLevel = aiService.evaluateSkillLevel(skillQAs);
            
            // 找出简历声称的级别
            String claimedLevel = findClaimedLevel(resumeData, skillName);
            
            // 计算差距
            report.addResult(skillName, claimedLevel, verifiedLevel, skillQAs.size(), correctRate);
        }
        
        return report;
    }
}
```

### 4.3 可配置简历模版

```sql
CREATE TABLE `resume_template` (
    `id`          BIGINT AUTO_INCREMENT PRIMARY KEY,
    `name`        VARCHAR(100) NOT NULL COMMENT '模版名称，如 Java后端模版',
    `direction`   VARCHAR(50)  NOT NULL COMMENT '适用方向：java_backend / frontend / fullstack',
    `sections`    JSON         NOT NULL COMMENT '章节定义（见下）',
    `is_active`   TINYINT DEFAULT 1,
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- sections 格式：
[
  {"key": "basicInfo",     "label": "基本信息",    "required": true,  "fields": [...]},
  {"key": "education",     "label": "教育经历",    "required": true,  "fields": [...]},
  {"key": "workExperience","label": "工作经历",    "required": false, "fields": [...]},
  {"key": "projects",      "label": "项目经验",    "required": true,  "fields": [...]},
  {"key": "skills",        "label": "技能标签",    "required": true,  "fields": [...]},
  {"key": "selfEvaluation","label": "自我评价",    "required": false, "fields": [...]}
]
```

模版的可配置化允许后台运营为不同岗位定义不同表单，而前端 `ResumeForm.vue` 根据 `sections` JSON 动态渲染。

## 5. 差异表：初级 vs 完善

| 维度 | 初级方案 | 完善方案 |
|---|---|---|
| **数据入口** | 上传解析 或 在线填写**二选一** | 上传后自动进入编辑确认页，双入口一体化 |
| **版本管理** | 每新增一条，is_active 标记 | 完整版本管理 + 版本对比 diff 页 |
| **简历模版** | 固定表单字段 | 管理后台可配置模版，前端动态渲染 |
| **出题策略** | 全量简历数据拼入 Prompt，无区分 | 按技能级别差异化出题（精通→hard，了解→easy） |
| **题目选择** | 纯 AI 决定 | AI + 数据库过滤，优先选技能标签匹配的题目 |
| **诊断增强** | 无 | 简历 vs 面试表现的技能验证报告 |
| **解析降级** | 重试 + 失败标注 | circuit breaker + 半结构化 regex 提取 |
| **文件格式** | PDF + TXT | PDF + DOCX + TXT + 图片 OCR（可选） |
| **管理后台** | 无 | 简历数据统计、技能分布聚合、模版管理 |
| **并发能力** | 串行解析 | 队列+异步解析（同一用户串行，不同用户并行） |
| **安全** | 基础 JWT 校验 | + 文件 MIME 校验 + 文件类型白名单 + 存储路径隔离 |
