# 架构设计与接口规范

## 1. 分层架构

```
┌──────────────────────────────────────────────────────────┐
│                    Presentation Layer                     │
│   ResumeUpload.vue  │  ResumeForm.vue  │  ResumeList.vue │
│   ResumeDetail.vue  │  VersionCompare.vue                │
└──────────────────────┬───────────────────────────────────┘
                       │ HTTP/SSE
┌──────────────────────▼───────────────────────────────────┐
│                     Controller Layer                      │
│   ResumeController        ResumeAdminController           │
│   POST /upload   GET /list   PUT /update   POST /create  │
└──────────────────────┬───────────────────────────────────┘
                       │
┌──────────────────────▼───────────────────────────────────┐
│                     Service Layer                         │
│  ┌─────────────────┐  ┌────────────────────────────┐     │

│  │ - 版本管理       │  │ - 重试与降级                │     │
│  │ - 激活切换       │  │ - Prompt管理               │     │
│  └────────┬────────┘  └────────┬───────────────────┘     │
│           │                    │                          │
│  ┌────────▼────────────────────▼───────────────────┐     │
│  │           External Infrastructure                 │     │
│  │  AIService(复用)  │  FileService / OSS          │     │
│  └─────────────────────────────────────────────────┘     │
└──────────────────────┬───────────────────────────────────┘
                       │
┌──────────────────────▼───────────────────────────────────┐
│                      Data Layer                           │
│   ResumeMapper  │  ResumeSessionMapper                   │
│   resume table  │  resume_session table                  │
└──────────────────────────────────────────────────────────┘
```

### 职责边界

| 层次 | 核心职责 | 禁止行为 |
|---|---|---|
| Controller | 参数校验、路由分发、状态码、统一响应 | 写业务逻辑、直接操作数据库 |
| Service | 业务编排、版本控制、AI 解析调度 | 操作 HttpServletRequest/Response |
| Mapper | 单一 SQL 映射 | 跨表复杂业务逻辑、循环查询 |
| Entity | 数据载体，ORM 映射 | 业务方法、非持久化字段过多 |
| Parser | AI Prompt 组装 + 结果反序列化 | 操作数据库、HTTP 调用（非 AIService） |

## 2. 状态机

### 2.1 简历生命周期

```
        [上传/创建]
            │
            ▼
      ┌──────────┐
      │ PARSEING │  ← 仅上传来源，在线填写的直接跳过此状态
      │ (status=0)│
      └─────┬────┘
            │
     ┌──────┴──────┐
     ▼              ▼
┌────────┐   ┌────────┐
│ ACTIVE │   │ FAILED │
│(status=1│   │(status=2│
│ +active│   │        │
└───┬────┘   └────────┘
     │
     │ [用户切到另一份]
     ▼
┌──────────┐
│ INACTIVE │
│ (不活跃)  │
└──────────┘
```

### 2.2 激活去激活规则

```
激活一份新简历时：
  1.  UPDATE resume SET is_active=0 WHERE user_id=? AND is_active=1
  2.  UPDATE resume SET is_active=1 WHERE id=?   ← 乐观锁
  3.  两语句在一个事务内

不允许：
  - 同时有两个 is_active=1（应用层约束）
  - 删除 is_active=1 的简历
  - 更新 is_active=1 简历的 parsed_data（须新建版本）
```

## 3. API 接口规范

### 3.1 通用规范

- 基础路径：`/api/resume`
- 响应格式（复用现有 `Result` 类）：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

- 分页请求参数：`page`(默认 1)、`size`(默认 10)
- 认证：所有接口需携带 JWT Token（复用 `UserInterceptor`）

### 3.2 接口列表

#### 简历上传

```
POST /api/resume/upload

Content-Type: multipart/form-data

参数：
  file: File  (必填, 支持 .pdf .docx .txt)

响应 200:
{
  "code": 200,
  "data": {
    "id": 1,
    "status": 0,
    "message": "文件上传成功，正在解析中"
  }
}

错误：
  400 - 文件类型不支持
  413 - 文件超过 10MB
```

#### 在线简历创建

```
POST /api/resume/create

Content-Type: application/json

请求体:
{
  "basicInfo": { ... },
  "skills": [ ... ],
  "projects": [ ... ],
  "education": [ ... ],
  "workExperience": [ ... ],
  "certifications": [ ... ],
  "selfEvaluation": "..."
}

响应 200:
{
  "code": 200,
  "data": {
    "id": 2,
    "version": 1,
    "status": 1,
    "isActive": true
  }
}

约束：
  请求体字段对应 parsed_data 格式，不做深度校验，
  由后续消费方容错。skills.level 若是枚举外的值，自动降级为"熟悉"。
```

#### 简历列表

```
GET /api/resume/list

查询参数：
  page: int (默认 1)
  size: int (默认 10)

响应:
{
  "code": 200,
  "data": {
    "records": [
      {
        "id": 1,
        "version": 3,
        "source": "upload",
        "status": 1,
        "isActive": true,
        "fileName": "张三_Java后端_2024.pdf",
        "fileSize": 245760,
        "applyPosition": "Java后端工程师",
        "workYears": 5,
        "skillCount": 12,
        "createTime": "2026-06-27 10:30:00"
      }
    ],
    "total": 5,
    "page": 1,
    "size": 10
  }
}
```

> `applyPosition`、`workYears`、`skillCount` 是从 `parsed_data` 中提取的展示字段，非独立列

#### 简历详情

```
GET /api/resume/{id}

响应:
{
  "code": 200,
  "data": {
    "id": 1,
    "userId": 42,
    "version": 3,
    "source": "upload",
    "status": 1,
    "isActive": true,
    "fileName": "张三_Java后端_2024.pdf",
    "fileUrl": "/files/resume/2026/06/xxx.pdf",
    "fileSize": 245760,
    "parsedData": { ... },    ← 完整 JSON
    "rawText": "...",
    "errorMsg": null,
    "createTime": "2026-06-27 10:30:00",
    "updateTime": "2026-06-27 10:32:00"
  }
}
```

> `parsedData` 是完整的 JSON 数据，前端需要自行解析渲染（或提交给 AI 消费方）

#### 简历编辑（基于已有版本创建新版本）

```
PUT /api/resume/{id}/update

Content-Type: application/json

请求体: (parsed_data 格式)
{
  "basicInfo": { ... },
  "skills": [ ... ],
  ...
}

响应:
{
  "code": 200,
  "data": {
    "id": 6,           ← 新版本 ID
    "version": 4,
    "source": "template",  ← 编辑后来源标记为 template
    "isActive": true
  }
}

设计约束：
  - 本接口不修改原记录，而是 INSERT 新版本
  - source 始终标记为 "template"（不管原始来源是什么）
  - 编辑后的新版本自动设为 is_active=1
```

#### 激活/切换简历

```
PUT /api/resume/{id}/activate

响应:
{
  "code": 200,
  "data": {
    "id": 6,
    "isActive": true
  }
}
```

#### 删除简历

```
DELETE /api/resume/{id}

响应:
{
  "code": 200,
  "message": "删除成功"
}

约束：
  - is_active=1 的简历拒绝删除，返回 400
  - 已绑定面试的简历拒绝删除，返回 400（需先解绑）
```

#### 重新解析（上传来源的失败重试）

```
POST /api/resume/{id}/reparse

响应:
{
  "code": 200,
  "data": {
    "id": 1,
    "status": 0,
    "message": "重新解析中"
  }
}

约束：
  - 仅 source=upload 的简历可重新解析
  - 重新解析会覆盖当前记录的 parsed_data 和 status
```

#### 获取解析进度（轮询）

```
GET /api/resume/{id}/status

响应:
{
  "code": 200,
  "data": {
    "status": 0,
    "errorMsg": null
  }
}

status: 0-解析中 1-完成 2-失败
前端每 3s 轮询，15s 超时提示
```

#### 绑定简历到面试

```
POST /api/resume/bind-session

请求体:
{
  "resumeId": 1,
  "sessionId": 42
}

响应:
{
  "code": 200
}
```

## 4. 异常码规范

| HTTP Status | code | message | 场景 |
|---|---|---|---|
| 400 | 40001 | 文件类型不支持 | 上传非 PDF/Word/图片/TXT |
| 400 | 40002 | 文件大小超过限制 (10MB) | 文件过大 |
| 400 | 40003 | 空文件 | 上传内容为空 |
| 400 | 40004 | 请先激活其他简历 | 删除活跃简历 |
| 400 | 40005 | 简历已绑定面试 | 删除已绑定的简历 |
| 404 | 40401 | 简历不存在 | ID 不存在 |
| 400 | 40006 | 仅上传来源可重新解析 | 非 upload 来源调用重解析 |
| 500 | 50001 | AI 解析服务不可用 | DeepSeek API 失败 |
| 500 | 50002 | 解析结果格式异常 | AI 返回无法解析的内容 |

## 5. 安全约束

| 约束 | 实现 |
|---|---|
| 用户只能操作自己的简历 | Controller 层从 JWT 提取 userId，与请求资源 owner 比对 |
| 文件上传类型白名单 | `FileTypeValidator` 校验 MIME 类型 + 扩展名 |
| 文件大小限制 | Spring `spring.servlet.multipart.max-file-size=10MB` |
| 文件存储隔离 | 用户文件放入 `/{userId}/` 子目录 |
| 解析内容无害化 | AIService 已内置 Prompt 注入防护（system prompt 约束） |
