# 初级实现方案（最小可用版）

## 1. 目标

**一句话**：用户上传一份简历或在线填写，系统能存下来并在面试时让 AI 读到。

## 2. 范围

| 包含 | 不包含 |
|---|---|
| PDF/TXT 上传 + AI 解析 | OCR / 图片解析 |
| 在线填写表单（基础版） | 版本对比 |
| 简历激活 / 切换 | 技能验证报告 |
| 简历绑定面试 | 简历优化建议 |
| 简单的简历列表 + 详情 | 管理后台统计 |

// Resume.java
@Data
@TableName("resume")
public class Resume {
    private Long id;
    private Long userId;
    private Integer version;
    private String source;       // upload / template
    private Integer status;      // 0-解析中 1-完成 2-失败
    private String errorMsg;
    private String rawText;
    
    @TableField(typeHandler = JacksonTypeHandler.class)
    private ResumeData parsedData;    // ← 核心，反序列化成 ResumeData VO
    
    private String fileName;
    private String fileUrl;
    private Long fileSize;
    private Integer isActive;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
}

// ResumeData.java — 入参/出参共用的 VO，与 parsed_data JSON 结构 1:1
@Data
public class ResumeData {
    private BasicInfo basicInfo;
    private List<Education> education;
    private List<WorkExperience> workExperience;
    private List<Project> projects;
    private List<Skill> skills;
    private List<String> certifications;
    private String selfEvaluation;
    // 只定义已知结构，不强制校验——未知字段 Jackson 默认忽略
}

@Data
public static class Skill {
    private String name;
    private String level;   // 精通/熟练/熟悉/了解
    private Integer years;
}

### Step 3: ResumeParserService（核心）

```java
@Service
public class ResumeParserService {

    private final AIService aiService;

    /**
     * 解析简历文本为结构化 ResumeData
     * @param rawText PDF/Word 等提取的纯文本
     * @return ResumeData （可能为 null）
     * @throws ParseException 3 次重试后仍失败
     */
    public ResumeData parse(String rawText) throws ParseException {
        String prompt = String.format("""
            你是一个专业的简历解析器。请从以下简历文本中提取结构化信息……

            ---
            %s
            ---
            """, rawText);

        try {
            return aiService.chatStructured(prompt, ResumeData.class);
        } catch (Exception e) {
            throw new ParseException("AI 解析失败", e);
        }
    }
}
```

### Step 4: FileService（文件提取）

```java
@Service
public class FileExtractService {

    /**
     * 从上传文件中提取纯文本
     */
    public String extractText(MultipartFile file, String extension) {
        return switch (extension.toLowerCase()) {
            case "pdf" -> extractFromPdf(file);
            case "docx" -> extractFromDocx(file);    // 可选
            case "txt" -> extractFromTxt(file);
            default -> throw new UnsupportedFileTypeException(extension);
        };
    }

    private String extractFromPdf(MultipartFile file) {
        // Apache PDFBox 3.x
        try (PDDocument doc = PDDocument.load(file.getInputStream())) {
            return new PDFTextStripper().getText(doc);
        }
    }
}
```

### Step 6: 面试绑定入口

修改 `InterviewController.create`，增加可选参数 `resumeId`：

```java
@PostMapping("/api/interview/create")
public Result create(@RequestBody InterviewCreateVO vo) {
    // 原有逻辑不变……
    if (vo.getResumeId() != null) {
        Resume resume = resumeService.getById(vo.getResumeId());
        // 把 parsed_data 写入 interview_context（Redis）
        interviewContextService.setResumeData(session.getId(), resume.getParsedData());
    }
}
```

在 `PromptBuilder` 中，如果上下文包含简历数据，在 System Prompt 中追加：

=== 面试者简历信息 ===

## 6. 初级 vs 完善对比

| 维度 | 初级 | 完善 |
|---|---|---|
| 入口 | 上传解析 / 在线填写 **二选一** 即可用 | 双入口 + 上传后可编辑 |
| 版本 | 仅保存最新版 | 完整版本管理 + 版本对比 |
| 面试出题 | 简历数据全量拼入 Prompt | 按技能层次差异化出题策略 |
| 诊断 | 无简历感知 | 简历 vs 表现对比报告 |
| 解析降级 | 重试 + 标注失败 | 重试 + circuit breaker + 半结构化提取 |
| 文件支持 | PDF + TXT | PDF + DOCX + TXT + 图片OCR（可选） |
| 前端 | 基础表单 | 可配置模版 + 拖拽排序 + 富文本编辑器 |

## 7. 验收标准

- [ ] 上传 PDF 简历 → AI 解析成功 → parsed_data 存库 → 列表可见
- [ ] 在线填写表单 → 保存成功 → 列表可见
- [ ] 只有 active 简历被面试使用
- [ ] 面试中 AI 回答与简历技能相关（而非通用问题）
- [ ] 删除非活跃简历 → 成功
- [ ] 删除活跃简历 → 被拒绝
